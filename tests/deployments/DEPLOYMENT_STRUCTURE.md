# Test Deployments Structure

This directory contains minimal Kubernetes manifests for testing the multi-cluster setup.

## Directory Structure

```
deployments/
├── app-cluster/           # Application cluster deployments
│   ├── budproxy.yaml     # BudProxy (TensorZero Gateway)
│   └── README.md
├── inference-cluster/     # Inference cluster deployments
│   ├── aibrix.yaml       # Placeholder AIBrix (nginx)
│   ├── aibrix-with-redis.yaml  # Static AIBrix with Redis
│   ├── build-aibrix.sh   # Build AIBrix from source
│   ├── AIBRIX_BUILD_INSTRUCTIONS.md
│   └── README.md
└── deploy-test-services.sh  # Deploy all test services
```

## Files Purpose

### Essential Files
- **Build Script**: `build-aibrix.sh` - Builds from source with Go auto-detection and Redis setup
- **YAML Files**: Provide different deployment options (placeholder, static)
- **Documentation**: README and build instructions explain usage and options

### Temporary Files (Not Tracked)
- `aibrix-local.yaml` - Generated by build scripts with current git hash

## Usage

1. **Quick Testing**: Use placeholder deployments
2. **Development**: Use build scripts for source changes
3. **Pre-built Images**: Use static YAML files

All files are essential for different testing scenarios.
